# PulseOn 0.1.0a4

This alpha release adds S3-compatible object-storage support for the DuckLake
data area while keeping catalog databases local. It also tightens the public
Python type surface for supported catalog backends.

## Scope

- `data_path` now accepts S3-compatible URIs such as
  `s3://bucket/prefix` for the DuckLake Parquet data area.
- S3 data paths can be configured through `<project>/.pulseon/config.toml` or
  explicit `pulseon.init(...)` keyword arguments. Explicit keywords override
  config-file values.
- S3 configuration is applied to the current DuckDB connection through
  connection-local HTTPFS/S3 settings and secrets.
- DuckDB and SQLite catalog backends both support S3-backed data paths.
- Local data-path behavior is preserved for writes, metric queries, terminal
  finalization, terminal Parquet visibility, and `flush_run_data(run_id)`.
- `catalog_path` remains local-only. Passing `catalog_path="s3://..."`
  continues to raise `InvalidConfigurationError`.
- The public Python type hints for `catalog_backend` now accept only
  `Literal["duckdb", "sqlite"]`; unsupported strings still fail at runtime
  through the existing validation path.

## MinIO Acceptance

Opt-in MinIO acceptance coverage now exercises S3-backed data paths for both
DuckDB and SQLite catalog backends. The acceptance path covers initialization,
metric writes, query visibility, terminal finalization, terminal Parquet
visibility, and `flush_run_data(run_id)`.

The tests require caller-provided environment variables and do not commit
credentials to source control:

- `PULSEON_MINIO_ENDPOINT`
- `PULSEON_MINIO_BUCKET`
- `PULSEON_MINIO_ACCESS_KEY_ID`
- `PULSEON_MINIO_SECRET_ACCESS_KEY`
- `PULSEON_MINIO_REGION` is optional and defaults to `us-east-1`.
- `PULSEON_MINIO_USE_SSL` is optional and defaults to `false`.

## Compatibility

- Python package version: `0.1.0a4`.
- Requires Python 3.11 or newer on CPython.
- Existing development stores may require a fresh local store. This alpha does
  not provide migration support for prior development catalogs.
- The public metric logging contract remains explicit-step
  `run.log(key, step, value)`.
- `data_path`, `catalog_backend`, `catalog_path`,
  `metric_queue_capacity`, and explicit S3 keyword arguments remain
  initialization options.
- `catalog_backend` remains runtime string-compatible for Python callers even
  though the public type hints are narrowed to supported literal values.

## Known Limits

- This is still an alpha release, not a stable compatibility commitment.
- S3 credential discovery through environment variables or AWS credential
  chains is deferred; this alpha requires explicit config-file or keyword
  credentials for S3 data paths.
- `catalog_path` is intentionally local-only.
- PostgreSQL shared-catalog support is deferred.
- SQLite multi-client lock parity and lock-file cleanup are deferred.
- Public debug-dump or verbose diagnostics APIs are deferred; ordinary
  diagnostics remain sanitized.
- Cloud, workspace hierarchy, config/tag filtering, built-in plotting, and AI
  Native features remain outside this release.
