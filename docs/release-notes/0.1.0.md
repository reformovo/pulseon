# PulseOn 0.1.0

PulseOn 0.1.0 promotes the frozen 0.1.0rc1 surface to the first non-alpha
release. The headless read surface, native storage boundary, and read-only CLI
are unchanged from the release candidate; no release blocker surfaced during
the RC window.

## Validation

Fresh packaged stores pass the DuckDB/SQLite and local/MinIO matrix, including
online and offline LTTB loading. Both catalogs pass process restart, sustained
writes, active-client reads, both terminal states, flush, and reopen.

## Known Limits

- An independent default-DuckDB reader may fail to attach or retain a stale
  snapshot while a writer is active. Use the writer client for live queries or
  open independent readers after shutdown; the equivalent SQLite path passes.
- LTTB is optional and platform-specific; the CLI may install it online, while
  offline use requires a compatible signed extension. The CLI stays read-only,
  and PostgreSQL, SQLite multi-writer parity, plotting, Web UI, and MCP remain
  deferred. Pre-1.0 APIs and stores have no compatibility promise.

## Verification

Release preparation on 2026-07-16 passed Rust formatting, `cargo check`,
Clippy with warnings denied, and Rust tests. The 0.1.0 development install,
Pyright, and ordinary pytest cases passed; opt-in MinIO cases skipped in the
ordinary suite. The release wheel and sdist built as `0.1.0`, and the
wheel passed the four-combination MinIO storage matrix plus both LTTB paths.
Packaged validation found the documented DuckDB reader limit and no other
workload failure.
